# 🔓 OCrackml — Analyse de fuites de données

Outil d'analyse et d'exploitation de fuites de données développé en **OCaml**.

---

## 🎯 Présentation

OCrackml est un projet d'analyse de fuites de données issues de différentes applications web. Il permet de croiser, comparer et craquer des mots de passe à partir de fichiers de fuites, en s'appuyant sur des fonctions de hashage et des structures de données adaptées.

---

## ✨ Fonctionnalités

- **Fusion de fuites** : regroupe les fuites d'une même application en éliminant les doublons (même pseudo et même mot de passe)
- **Détection multi-fuites** : identifie si un même login est présent dans plusieurs fuites et compare les mots de passe associés
- **Analyse de hashes** : repère les mots de passe hachés communs à plusieurs fuites et les logins qui leur sont liés
- **Recherche par mot de passe** : retrouve les couples `(application, login)` correspondant à une liste de mots de passe en clair
- **Crackage de mots de passe** : retrouve les mots de passe en clair à partir de leurs hashes

Les données sont lues depuis des fichiers texte et les mots de passe sont comparés via leur hash.

---

## 🗂️ Structure du projet

```c
.
├── main.ml                          # Point d'entrée, exécution de l'analyse complète
├── crack.ml                         # Logique de crackage et d'analyse
├── slogram01.txt, slogram02.txt     # Exemples de fichiers de fuites (application Slogram)
├── tetedamis01.txt, tetedamis02.txt # Exemples de fichiers de fuites (application Tetedamis)
└── french_passwords_top20000.txt    # Liste de mots de passe en clair
```

---

## 📋 Fonctions principales

| Fonction | Description |
|----------|-------------|
| `merge_file` | Fusionne plusieurs fichiers d'une même application et supprime les doublons |
| `check_same_login` | Identifie si un même login est présent dans plusieurs fuites et si les mots de passe sont identiques |
| `check_same_hash` | Détecte les mots de passe hachés communs et les logins associés |
| `valid_logins` | À partir d'une liste de mots de passe en clair, récupère les couples `(application, login)` correspondants |
| `crack_passwords` | Retrouve les mots de passe en clair associés aux hashes |
| `crack` | Fonction principale exécutant l'analyse complète sur les fichiers des applications et les mots de passe |

## 🔧 Prérequis

OCaml et OPAM installés sur votre machine, ainsi que les bibliothèques suivantes :

```bash
opam install cryptokit base64
```

---

## ▶️ Compilation et exécution

1. Placer les fichiers de fuites dans le même dossier que le programme :
   ```
   slogram01.txt, slogram02.txt, tetedamis01.txt, tetedamis02.txt, ...
   ```

2. Placer la liste des mots de passe en clair dans le même dossier :
   ```
   french_passwords_top20000.txt
   ```

3. Lancer le programme :
   ```bash
   ocaml main.ml
   ```

Le programme retourne un tableau de triplets :
```
(application, pseudo, mot_de_passe_en_clair)
```
et indique le nombre total de comptes craqués.
